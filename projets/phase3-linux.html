Serveur Linux, supervision et sauvegarde

Phase 3 — Ce qui fait la différence

⏳
Phase à venir, après la Phase 2. C'est ici que je sors du tout-Microsoft pour découvrir Linux et la supervision.

Contexte

L'infrastructure Windows tourne. Il me reste à mettre en place la supervision (vérifier que tout fonctionne sans devoir le contrôler à la main), la sauvegarde, et un petit serveur web Linux pour héberger un site interne.

Objectifs

  • Installer et configurer Debian 12 (SRV-LNX01)
  • Déployer un serveur web (Apache ou Nginx)
  • Mettre en place Zabbix pour superviser l'ensemble de l'infrastructure
  • Configurer un plan de sauvegarde avec Windows Server Backup

Démarche prévue

  • Créer la VM SRV-LNX01 (Debian 12, IP 192.168.10.11)
  • Installer un serveur web (Apache) pour héberger une page d'accueil interne
  • Déployer Zabbix sur SRV-LNX01 et installer les agents sur les autres machines pour superviser l'état général (CPU, disque, services)
  • Configurer Windows Server Backup pour sauvegarder les partages et l'état système du DC sur un disque dédié
  • Tester la restauration pour vérifier que la sauvegarde fonctionne

Page enrichie au fur et à mesure de la réalisation.

Compétences mobilisées

  • Installer et administrer un serveur Linux
  • Déployer un serveur web
  • Mettre en place une solution de supervision
  • Planifier et configurer des sauvegardes
