Serveur Linux, supervision et sauvegarde

Phase 3 — Ce qui fait la différence

⏳
Phase à venir — Démarrera une fois les services Windows essentiels en place (Phase 2). C'est ici que je sors du tout-Microsoft pour ouvrir l'infra à Linux et à la supervision.

Contexte

L'infrastructure Windows de BleuesLatitudes est opérationnelle. Il faut maintenant assurer la disponibilité des services avec une supervision proactive, proposer un hébergement web interne, et mettre en place un plan de sauvegarde pour protéger les données critiques (devis, comptabilité).

Objectifs

  • Installer et configurer Debian 12 (SRV-LNX01)
  • Déployer un serveur web (Apache ou Nginx)
  • Mettre en place Zabbix pour superviser l'ensemble de l'infrastructure
  • Configurer un plan de sauvegarde avec Windows Server Backup

Démarche prévue

  • Création de la VM SRV-LNX01 (Debian 12, 1 Go RAM, IP 192.168.10.11), durcissement de base : SSH par clé, désactivation root, mises à jour automatiques
  • Installation d'Apache (ou Nginx) pour héberger un site interne — page d'accueil minimale présentant les services de l'entreprise
  • Déploiement de Zabbix Server + agent sur les machines Windows et Linux, création de tableaux de bord et alertes (CPU, espace disque, services AD/DNS)
  • Mise en place de Windows Server Backup : sauvegarde planifiée des partages de SRV-FS01 et de l'état système des contrôleurs de domaine, vers un disque dédié
  • Test de restauration partielle — étape souvent oubliée mais qui valide réellement le plan de sauvegarde

Cette section sera enrichie avec les captures d'écran et les choix techniques argumentés au fur et à mesure de la réalisation.

Compétences mobilisées

  • Installer et administrer un serveur Linux
  • Déployer un serveur web
  • Mettre en place une solution de supervision
  • Planifier et configurer des sauvegardes
