Mise en place du domaine Active Directory

Phase 1 — Les fondations de l'infrastructure BleuesLatitudes

🚧
Phase en cours. Le domaine bleueslatitudes.local est créé. Un point à débloquer avant de continuer : l'accès administrateur du domaine (résolution prévue via le mode DSRM, détails plus bas). Une fois ce point réglé, je passe à la création des OU, groupes et utilisateurs.

Contexte

Côté infrastructure, BleuesLatitudes part de zéro : 25 postes en groupe de travail, aucun compte centralisé, aucune sauvegarde, aucune supervision. La première brique à poser est un domaine Active Directory. C'est lui qui conditionne tout le reste : les permissions sur les fichiers (Phase 2), les GPO, le partage centralisé.

Objectifs

  • Installer et configurer Windows Server 2025 (SRV-DC01)
  • Configurer l'adressage IP fixe sur le réseau 192.168.10.0/24
  • Déployer le rôle Active Directory Domain Services
  • Promouvoir le serveur en contrôleur de domaine — forêt bleueslatitudes.local
  • Créer la structure organisationnelle (OU par service)
  • Créer les comptes utilisateurs et groupes
  • Joindre un poste client (PC-01) au domaine

Environnement technique

  • Hyperviseur : Hyper-V (Windows 11 Pro, hôte 24 Go DDR4)
  • Switch virtuel : BLEUESLAT-NET (Private)
  • VM SRV-DC01 : Génération 2 — 2 vCPU — 2 Go RAM — disque 60 Go — Windows Server 2025 Standard (Expérience utilisateur)
  • Adressage : 192.168.10.1/24, passerelle non définie (lab isolé), DNS 127.0.0.1

Avancement détaillé

✅ Étape 1 — Création de la VM SRV-DC01

VM créée sous Hyper-V (Génération 2) à partir de l'ISO Windows Server 2025. J'ai choisi la version 2025 plutôt que la 2022 pour une raison simple : autant me former sur la version la plus récente, qui sera celle que je rencontrerai en entreprise.

✅ Étape 2 — Réseau et identité

IP fixe en 192.168.10.1/24, DNS pointé sur 127.0.0.1 (le serveur sera lui-même DNS une fois promu). J'ai renommé la machine en SRV-DC01 avant la promotion AD. Un point que j'ai retenu : renommer un contrôleur de domaine après coup demande des manipulations délicates, autant le faire dans le bon ordre dès le départ.

✅ Étape 3 — Installation du rôle AD DS

Installation du rôle AD DS depuis le Gestionnaire de serveur, puis promotion du serveur en contrôleur de domaine. Nouvelle forêt avec bleueslatitudes.local comme domaine racine, niveau fonctionnel Windows Server 2025. J'ai aussi défini le mot de passe DSRM (Directory Services Restore Mode), qui sert de filet de sécurité si l'accès au compte administrateur du domaine pose un jour problème. Comme on va le voir juste après, ce filet sert plus tôt que prévu.

⚠️ Point bloquant — Accès admin du domaine

C'est ici que je rencontre la première vraie difficulté : impossible de me connecter en tant qu'administrateur du domaine, le mot de passe est refusé. Deux causes possibles : soit le compte est verrouillé suite à plusieurs tentatives, soit ce que j'ai noté ne correspond pas exactement à ce que j'ai saisi lors de la promotion.

Plan pour débloquer : redémarrer en mode DSRM, réinitialiser le compte Administrator via net user Administrator [nouveau-mdp], puis revenir en mode normal. C'est précisément le cas d'usage pour lequel le mot de passe DSRM existe — sans lui, j'aurais dû reprendre la VM à zéro.

⏳ Étape 4 — OU, groupes et utilisateurs (à venir)

Une fois l'accès administrateur récupéré, je structure l'annuaire avec une OU par service (Direction, Bureau d'études / Commercial, Administratif), je crée les groupes de sécurité correspondants et les comptes des 25 employés. Cette étape est essentielle car tout ce qui vient après s'appuie dessus : les permissions sur le serveur de fichiers en Phase 2, les GPO ciblées, etc.

⏳ Étape 5 — Joindre PC-01 au domaine (à venir)

Création de la VM PC-01, configuration réseau, jonction au domaine. L'objectif : vérifier qu'un poste client peut s'authentifier sur le DC avec un compte AD — la preuve concrète que tout fonctionne bout en bout.

Ce que je retiens

  • AD et DNS sont indissociables : un contrôleur de domaine est obligatoirement serveur DNS de sa zone, ce n'est pas optionnel.
  • Promouvoir un serveur en contrôleur de domaine est une opération qui ne se défait pas à la légère, d'où l'utilité du mot de passe DSRM (que j'ai pris soin de noter correctement, lui).
  • L'ordre des opérations compte : adressage IP → renommage → installation du rôle → promotion. Inverser ces étapes oblige à des manipulations correctives ensuite, ce n'est jamais propre.
  • Le niveau fonctionnel (forêt, domaine) détermine les fonctionnalités disponibles et les versions de DC tolérées dans la forêt. Important si on envisage de faire cohabiter plusieurs versions de Windows Server par la suite.

Compétences mobilisées

  • Installer et configurer un serveur Windows
  • Administrer Active Directory
  • Gérer les utilisateurs et les groupes
  • Configurer l'adressage réseau
  • Mettre en place une convention de nommage et un plan d'adressage
