Mise en place du domaine Active Directory

Phase 1 — Les fondations de l'infrastructure BleuesLatitudes

🚧
Phase en cours — Domaine bleueslatitudes.local créé. Point bloquant à résoudre : accès administrateur du domaine (résolution prévue via mode DSRM, voir section Avancement détaillé). Étape suivante : création des OU, groupes et utilisateurs des 3 services.

Contexte

BleuesLatitudes fonctionne en groupe de travail : chaque poste est indépendant, les employés n'ont pas de compte centralisé, et la gestion informatique est inexistante. L'objectif est de mettre en place un domaine Active Directory pour centraliser la gestion des identités et des accès, et préparer le terrain pour les phases suivantes (DHCP, partage de fichiers, GPO).

Objectifs

  • Installer et configurer Windows Server 2025 (SRV-DC01)
  • Configurer l'adressage IP fixe sur le réseau 192.168.10.0/24
  • Déployer le rôle Active Directory Domain Services
  • Promouvoir le serveur en contrôleur de domaine — forêt bleueslatitudes.local
  • Créer la structure organisationnelle (OU par service)
  • Créer les comptes utilisateurs et groupes
  • Joindre un poste client (PC-01) au domaine

Environnement technique

  • Hyperviseur : Hyper-V (Windows 11 Pro, hôte 24 Go DDR4)
  • Switch virtuel : BLEUESLAT-NET (Private)
  • VM SRV-DC01 : Génération 2 — 2 vCPU — 2 Go RAM — disque 60 Go — Windows Server 2025 Standard (Expérience utilisateur)
  • Adressage : 192.168.10.1/24, passerelle non définie (lab isolé), DNS 127.0.0.1

Avancement détaillé

✅ Étape 1 — Création de la VM SRV-DC01

Création d'une VM Génération 2 sous Hyper-V à partir de l'ISO Windows Server 2025. Le choix de la version 2025 (au lieu de 2022) répond à une logique simple : se former sur la version la plus récente, celle que je rencontrerai en alternance.

✅ Étape 2 — Configuration réseau et identité système

Adresse IP fixée à 192.168.10.1/24, DNS pointé sur 127.0.0.1 (le serveur sera son propre DNS une fois promu). Renommage de la machine en SRV-DC01 (convention de nommage du projet) avant la promotion AD — étape importante : on ne renomme pas un contrôleur de domaine après coup sans précautions.

✅ Étape 3 — Installation du rôle AD DS

Installation du rôle Active Directory Domain Services via le Gestionnaire de serveur, puis promotion en contrôleur de domaine. Création d'une nouvelle forêt avec le domaine racine bleueslatitudes.local — niveau fonctionnel forêt et domaine Windows Server 2025. Définition du mot de passe DSRM (Directory Services Restore Mode), qui servira à récupérer la main en cas de problème.

⚠️ Point bloquant — Accès administrateur du domaine

Après promotion, la connexion en tant qu'administrateur du domaine est refusée (mot de passe incorrect). Hypothèses : verrouillage temporaire après plusieurs tentatives, ou écart entre mot de passe noté et mot de passe réellement saisi. Plan de reprise prévu : redémarrage en mode DSRM puis réinitialisation du compte Administrator via net user Administrator [nouveau-mdp], retour en mode normal. C'est précisément à ça que sert le mot de passe DSRM défini à l'étape 3.

⏳ Étape 4 — Création des OU, groupes et utilisateurs (à venir)

Prochaine étape : structurer l'annuaire avec une OU par service (Direction, Bureau d'études / Commercial, Administratif), créer les groupes de sécurité associés et les comptes des 25 employés. Cette structuration conditionne la suite : les permissions sur le serveur de fichiers (Phase 2) et les GPO ciblées seront appliquées via ces OU et groupes.

⏳ Étape 5 — Jonction de PC-01 au domaine (à venir)

Création de la VM PC-01, configuration réseau, et jonction au domaine pour valider l'authentification centralisée.

Ce que j'apprends à cette étape

  • Le DNS est indissociable d'AD — un contrôleur de domaine est obligatoirement serveur DNS de la zone.
  • La promotion en contrôleur de domaine est une opération à sens unique : on ne revient pas en arrière à la légère, d'où l'intérêt du mot de passe DSRM.
  • L'ordre des opérations compte : adressage IP → renommage → installation du rôle → promotion. Inverser ces étapes oblige à des manipulations correctives ensuite.
  • Les niveaux fonctionnels (forêt, domaine) déterminent les fonctionnalités disponibles et les versions de DC tolérées dans la forêt.

Compétences mobilisées

  • Installer et configurer un serveur Windows
  • Administrer Active Directory
  • Gérer les utilisateurs et les groupes
  • Configurer l'adressage réseau
  • Mettre en place une convention de nommage et un plan d'adressage
